Технология доступа к данным ADO.NET
Введение

Технология ADO связывает способы программирования платформы .NET(C#) и систему
управления базой данных (СУБД), т.е. даёт доступ к данным с БД в программный код.

Технология содержит универсальные способы, которые подходят для разных СУБД
(PostgresSQL, MSSQL/SQL Server, MySQL, Oracle XE, и тд.)

Отличие разных СУБД в том, что используются разные типы данных.

C#                                         DB
SELECT 1  ------------------------------>  1
          <------------------------------
              ADO        Provider

Provider(Driver) - драйвер, коннектор - спец. программы для перевода общих команд ADO
в команды СУБД.

Создание проекта:
1. Новый проект WPF
2. Создаём БД : ПКМ по проекту - Add - new item - [Data] Service-based DB
3. Двойной клик по БД - открывается Server Explorer и показывает БД.
4. ПКМ - Properties - в окне копируем строку "Connection String"
   Data Source=(LocalDB)\MSSQLLocalDB;AttachDbFilename=E:\C#\ШАГ_ADO_NET\first\first\stepDB.mdf;Integrated Security=True
5. Вставляем это значение как константу Программы (App.xaml.cs)
6. В зависимости от выбранного СУБД (MSSQL) добавляем провайдера.
   Как правило это делается через NuGet. Для MSSQL провайдер - Microsoft.Data.SqlClient

Работа с БД:
1. Подключение к БД - основной объект, который обеспечивает передачу команд к БД и результаты её
   выполнения к программе.
   !!! Особенность .Net в том, что создание объекта-подключения:
       SqlConnection connection = new SqlConnection(App.ConnectionString);
       не подключает БД, только создаёт объект.
   Для открытия подключения нужна команда:
   connection.Open();

   !!! Подключение к БД принадлежит к unmanage(неуправляемых) ресурсов, поэтому, их
   нужно удалять и освобождать способами:
   1. блок using
   2. метод Dispose()
2. Выполнение запросов - передача текста SQL к СУБД.

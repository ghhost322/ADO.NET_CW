Выполнение запросов. Обработка и отображение результатов.

Для отправки запросов к СУБД использ. класс SqlCommand
Для запросов разного типа есть несколько методов:
.ExecuteNonQuery - для запросов, которые не имеют результатов (все, кроме SELECT)
.ExecuteScalar - для запросов с одним (скалярным) результатом (SELECT COUNT(*) FROM ...)
.ExecuteReader - для запросов с табличным результатом

Особенности СУБД: когда выполняется запрос SELECT СУБД создаёт правило, итератор,
который может строку за строкой возвращать результат.

     ADO                                          DBMS
SELECT * FROM Groups  ------------------->  Компиляция запроса и создания правила
     Reader           <-------------------  итератор

Reader.Read()  --------------------------->  итератор.Next()
               <---------------------------  строка результата
У объекта Reader заполнены данные очередной строки результата.

Reader.Read()  --------------------------->  итератор.Next()
               <---------------------------  строка результата
следующая строка данных

index     title
  0         Id
  1         Name
  2         Description
  3         Picture
Как получить данные из Reader?
a) через Get-тер по индексу поля:
   reader.GetGuid(0)  -- Id as Guid
   reader.GetString(1)  -- Name as string
б) через Get-тер по имени поля:
   reader.GetString("Description")
в) через индексатор (не рекомендуется, поскольку возвращает object - не типизирует):
   reader["Picture"]

!!! После получения данных Reader его нужно закрывать, или использ. блок using.
    Иначе след. команды будут завершаться исключением "Есть не закрытый DataReader"

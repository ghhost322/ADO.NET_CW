Принцип создания объектов доступа к данным (DAO) который требует создания 4 групп методов,
предназначенных для CRUD операций.

Операция DELETE:
У реляционных данных данная операция очень небезопасна.
Поэтому никогда данные из БД не удаляем!!!

Практично использ. подход "пометки" данных, которые удалены (без реального их удаления).
Например, просто поле с датой удаления.



Подготовленные (параметризованные) запросы
Технология выполнения запросов с отдельными параметрами
Проблема - когда данные вставляются в код (запрос) они могут исказить этот запрос
Например:
$"INSERT INTO User VALUES(N'{Name}', ...)";
Name = "Лук`ян"
INSERT INTO User VALUES(N'Лук`ян', ...)";  --  SQL Error
